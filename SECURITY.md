# Security Policy

## Supported Versions

Security fixes will be applied to the current main branch.

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive reports.

Send a private report to the maintainers through the preferred GitHub security flow, or contact the repository owner directly if that flow is not available.

Include:

- A short summary of the issue
- Steps to reproduce
- Impact assessment
- Any suggested fix or mitigation

## Response Expectations

Maintainers should acknowledge confirmed reports and work toward a fix before public disclosure.
